IAR Embedded Workbench IDE and Code Composer Studio Users

IAR Embedded Workbench KickStart IDE and Code Composer Studio (CSS) are alternative development tools to the mpyEditor. They both use the C programming language to program the MSP430.  These tools are more complicated tools intended for professional electronic engineers.

If you use IAR  you may want to switch from using IAR to the mpyEditor,  or even switch from the mpyEditor to IAR.  Whichever software you use you will need to update the drivers in order to communicate with the MSP430.

To change from IAR to mpyEditor

The mpyEditor uses two drivers:

  • MSP430 Application UART
  • libusb-win32

If you want to use the mpyEditor the easiest way to install the drivers is start the mpyEditor and click the Install button.  (More details and other ways of installing the drivers can be found here). You may need to restart your computer before the driver is recognized, or you may need to unplug your Launchpad board and re-insert it, this will force the driver to be re-loaded.

You can now start the mpyEditor, and plug in the Launchpad

To change from mpyEditor to the IAR

The IAR uses a single driver:

  • MSP430 Application UART

This driver is the same driver as used by the mpyEditor. However the libusb-win32 driver is not used for the IAR setup, and it needs to be removed.

To remove the libusb-win32 driver:

  • Close the mpyEditor
  • Close the IAR tool
  • Plug in the Launchpad
  • Open the Device Manager
  • On Windows 7 Right Click on ‘My Computer -> Properties -> Device Manager
  • On Windows XP Right Click on My Computer -> Properties -> Hardware tab -> Device Manager
  • In the Device Manager window find the line with libusb-win32 devices,  click on the ‘+’ to expand it. Directly underneath it there should be a line with USB Human Interface Device (Interface 1)
  • Right Click on the USB Human Interface Device (Interface 1) and select Unintstall this will remove the driver.
  • Unplug the Launchpad.
  • You may need to restart your computer before the new driver setup is recognized, or you may need to unplug your Launchpad board and re-insert it, this will force the driver to be re-loaded.

Start up the IAR tool and configure it to use the correct Launchpad MPS430 chip, and you should now be able to plug in the Launchpad and program it using the Download and Debug button.

If this fails then look at the Device Manager and check that the libusb-win32 device is not present, if it is repeat the above instructions to remove it.
