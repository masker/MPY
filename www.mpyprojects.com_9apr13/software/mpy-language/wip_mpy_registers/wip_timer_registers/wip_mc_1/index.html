wip_MC_1

The Digital IO Functions provide basic commands to read and write digital values to the IO pins of the microcontroller.

When reading and writing values to the microcontroller pins a value of 0 will mean a voltage of 0v at the pin. A value of 1 will mean a voltage of 3.6v at the pin, or the power supply voltage connected to the MSP430 chip.

In order the use an input or output pin it must first be configured using one of the  following direction commands

  • dirinp( portpin )   sets the pin direction to be an input (pin is floating)
  • dirinpu( portpin )   sets the pin direction to be an input (with pullup resistor)
  • dirinpd( portpin )   sets the pin direction to be an input (with pulldown resistor)
  • dirout ( portpin )   sets the pin direction to be an output.

Then to read and write to the pins use:

  • value = inp( portpin )  reads the input pin value ( a 0 or a 1)
  • out( portpin , value )  sets the output value to value  (0v or 3.6v)
  • out0( portpin)  set the output to 0  (0v)
  • out1( portpin)  set the outpu to 1 (3.v

The mpy program will initialize all input pins to be inputs with a pulldown resistor (which is the same as the dirinpd( ) command) This ensures that all pins will be pulled to 0v when the Micorocontroller is first started.

Example:

define_micro('msp430g2553')    # <<< Change >>>  tell mpyEditor which msp430 to program
dirout(P1_0)                   # Set port P1_0 as an output (this is the Launchpad red led)
dirinpd(P1_3)                  # Set port P1_3 as an input with pull down resistor (SW2)
while 1:                       # do forever
   switch_input = inp(P1_3)    # read the input from pin P1_3
   out(P1_0, switch_input)     # set the output pin P1_0 (red led) to the switch value

This example will read the input switch on P1_3 and depending on whether it is pressed or not it will light the red led.

Note input pins which are configured using the dirinp( ) command will be floating if they are not connected to anything. This means that the voltage at the pin will ‘float’ around and will probably pick up stray interference. This can be a problem and cause the microcontroller to draw excessive current and cause other bad effects. So in general it is a good idea to use the dirinpu( ) or dirinpd( ) commands when setting up a pin as an input.
